## 1. 概述

### 1.1 算法背景

边界层网格生成是计算流体力学（CFD）数值模拟中的关键技术，用于捕捉物面附近的粘性效应。层推进法（Advancing Layers Method, ALM）是一种高效的边界层网格生成方法，通过从物面向外逐层推进生成结构化网格。

### 1.2 算法特点

**层推进算法（Adlayers2）**：
- 从物面边界逐层向外推进生成边界层网格
- 支持几何增长（geometric）的网格高度分布
- 自动生成四边形/三角形混合网格
- 支持多部件（multi-part）边界层生成
- 支持 match 边界条件处理

**多方向推进算法（Multi-Direction）**：
- 解决凸角处网格挤压问题
- 在凸角处分裂多个推进方向
- 创建虚拟点（virtual point）和虚拟阵面
- 实现凸角处网格的光滑过渡
- 支持串线追踪（strandline tracking）

### 1.3 适用范围

- 二维粘性 CFD 网格生成
- 复杂外形边界层网格
- 多部件组合体边界层
- 凸角/凹角处网格光滑过渡

## 3. 层推进算法

### 3.1 主推进流程

#### 3.1.1 总体流程（generate_elements）

```python
def generate_elements(self):
    """生成边界层网格"""
    for i, part in enumerate(self.part_params):
        if part.part_params.PRISM_SWITCH != "wall":
            continue
        
        # 设置当前推进部件参数
        self.set_current_part(part)
        
        self.ilayer = 0
        
        # 初始化多方向管理器
        if self.multi_direction:
            self.multi_direction_manager = MultiDirectionManager(self)
        
        # 单层推进循环
        while self.ilayer < self.max_layers and self.num_prism_cap > 0:
            self._process_single_layer(i, num_parts)
            self.ilayer += 1
    
    # 后处理：消除虚拟点
    if self.multi_direction and self.multi_direction_manager:
        self.multi_direction_manager.post_process()
    
    # 构建非结构网格
    self.construct_unstr_grid()
    
    return self.unstr_grid, self.all_boundary_fronts
```

#### 3.1.2 单层推进流程（_process_single_layer）

```python
def _process_single_layer(self, part_index, num_parts):
    """执行单层推进完整流程"""
    start_cell_idx = self.num_cells
    
    # 阶段 1：基础几何与方向
    self.prepare_geometry_info()
    
    # 阶段 2：多方向流程（初始化→方向光滑→步长缩放）

```

### 3.2 几何准备阶段

#### 3.2.1 阵面几何计算（prepare_geometry_info）

```python
def prepare_geometry_info(self):
    """准备几何信息"""
    # 1. 计算阵面几何属性（法向、方向、长度等）
    self.compute_front_geometry()
    
    # 2. 计算节点初始推进方向
    self.compute_point_normals()
    
    # 3. Laplacian 光滑节点推进方向
    self.laplacian_smooth_normals()
```

**关键步骤**：
1. **阵面法向计算**：根据阵面端点坐标计算单位法向量
2. **节点推进方向**：基于相邻阵面法向的加权平均
3. **方向光滑**：Laplacian 光滑提高方向场连续性

#### 3.2.2 节点推进方向计算（compute_point_normals）

**算法原理**：

对于物面上的节点，推进方向由相邻两个阵面的法向决定：

```python
# 相邻阵面法向
normal1 = np.array(front1.normal)
normal2 = np.array(front2.normal)

# 初始推进方向（法向量平均）
avg_direction = (normal1 + normal2) / 2.0
norm = np.linalg.norm(avg_direction)
avg_direction /= norm

# 加权迭代光滑
w1 = self.relax_factor  # 松弛因子（默认 0.2）
wf1 = 0.5  # 初始权重
wf2 = 0.5

for iteration in range(max_iterations):
    # 加权平均
    new_direction = (1 - w1) * old_direction + w1 * (
        wf1 * normal1 + wf2 * normal2
    )
    
    # 根据夹角偏差动态调整权重
    dot_product1 = np.dot(new_direction, normal1)
    dot_product2 = np.dot(new_direction, normal2)
    angle1 = np.arccos(dot_product1)
    angle2 = np.arccos(dot_product2)
    avg_angle = (angle1 + angle2) / 2
    
    df1 = abs(angle1 - avg_angle)
    df2 = abs(angle2 - avg_angle)
    
    # 更新权重：夹角偏差越小，权重越大
    wf1_bar = wf1 * (1 - df1 / (avg_angle + epsilon))
    wf2_bar = wf2 * (1 - df2 / (avg_angle + epsilon))
    wf1 = wf1_bar + wf1 * (1 - (wf1_bar + wf2_bar))
    wf2 = wf2_bar + wf2 * (1 - (wf1_bar + wf2_bar))
```

**特殊情况处理**：
- **Match 边界节点**：直接使用 `marching_vector`
- **相邻阵面不足 2 个**：通常为 match point，不计算推进方向
- **一侧为 interior 阵面**：将另一侧 wall 阵面法向作为推进方向
- **多方向凸点**：保持方向列表，不参与此阶段计算

#### 3.2.3 Laplacian 光滑（laplacian_smooth_normals）

**算法原理**：

基于邻居节点方向的加权平均，距离越近权重越大：

```python
for iteration in range(self.smooth_iterions):  # 默认 3 次迭代
    for node_elem in self.front_node_list:
        # 收集邻居节点方向
        summation = np.zeros_like(current_dir)
        total_weight = 0.0
        
        for neighbor in node_elem.node2node:
            # 距离权重：距离越近影响越大
            distance = np.linalg.norm(current_pos - neighbor_pos)
            weight = 1.0 / distance
            
            summation += weight * neighbor_dir
            total_weight += weight
        
        # 加权平均
        new_direction = (1 - relax_factor) * current_dir + \
                       relax_factor * (summation / total_weight)
        
        # 归一化
        new_direction /= np.linalg.norm(new_direction)
```

**参数说明**：
- `smooth_iterions`：光滑迭代次数（默认 3）
- `relax_factor`：松弛因子（默认 0.2，0-不光滑，1-完全光滑）

### 3.3 推进距离计算

#### 3.3.1 几何增长模型（calculate_marching_distance）

```python
def calculate_marching_distance(self):
    """计算节点推进距离"""
    if self.growth_method == "geometric":
        # 几何增长公式
        first_height = self.first_height  # 第一层高度
        growth_rate = self.growth_rate    # 增长率（默认 1.2）
        self.current_step_size = first_height * growth_rate**self.ilayer
    
    # 应用局部步长因子（多方向推进）
    for front in self.current_part.front_list:
        for node in front.node_elems:
            if self.multi_direction:
                node.marching_distance = self.current_step_size * node.local_step_factor
            else:
                node.marching_distance = self.current_step_size
```

**增长公式**：
```
h_i = h_1 * r^(i-1)
```
其中：
- `h_i`：第 i 层高度
- `h_1`：第一层高度（first_height）
- `r`：增长率（growth_rate，默认 1.2）
- `i`：层数索引（从 1 开始）

**局部步长因子** `local_step_factor`：
- 用于调整凸角/凹角处的推进步长
- 计算公式：`1 - sign(θm) * |θm| / π`
- 凸角（θm > 0）：因子 < 1，减小步长
- 凹角（θm < 0）：因子 > 1，增大步长

### 3.4 阵面推进（advancing_fronts）

#### 3.4.1 推进流程

```python
def advancing_fronts(self):
    """逐个阵面推进生成单元"""
    new_interior_list = []   # 新增 interior 阵面
    new_prism_cap_list = []  # 新增 prism-cap 阵面
    
    for front in self.current_part.front_list:
        if front.bc_type == "interior":
            new_interior_list.append(front)
            continue
        
        if front.early_stop_flag:
            new_prism_cap_list.append(front)
            continue
        
        # 处理零长度虚拟阵面
        if self._advance_virtual_front(front, ...):
            continue
        
        # 检查相邻阵面层数差
        if self._check_neighbor_layer_difference(front):
            front.early_stop_flag = True
            new_prism_cap_list.append(front)
            continue
        
        # 创建新单元和阵面
        (new_cell, alm_front, new_front1, new_front2, 
         new_node_generated, new_cell_nodes) = self.create_new_cell_and_front(front)
        
        # 几何检查（质量、长宽比、层数）
        if self.geometric_checker(front, new_cell):
            front.early_stop_flag = True
            new_prism_cap_list.append(front)
            continue
        
        # 邻近检查
        check_fronts = [alm_front, new_front1, new_front2]
        if self.proximity_checker(front, check_fronts):
            front.early_stop_flag = True
            new_prism_cap_list.append(front)
            continue
        
        # 更新节点、单元、阵面
        self._update_data_structure(...)
    
    # 更新下一层阵面列表
    self.num_prism_cap = len(new_prism_cap_list) - num_old_prism_cap
    self.current_part.front_list = new_prism_cap_list + new_interior_list
```

#### 3.4.2 新单元创建（create_new_cell_and_front）

**几何示意**：
```
节点推进前：
  node0 *--------* node1
         front (prism-cap)

节点推进后：
  node0 *--------* node1
        |        |
        |  quad  |
        |        |
  node2 *--------* node3
  
新阵面：
  - alm_front: node2 -> node3 (prism-cap)
  - new_front1: node0 -> node2 (interior)
  - new_front2: node3 -> node1 (interior)
```

**核心代码**：
```python
def create_new_cell_and_front(self, front):
    # 推进生成新节点
    new_cell_nodes = front.node_elems.copy()
    
    for i, node_elem in enumerate(front.node_elems):
        if node_elem.corresponding_node is None:
            # 计算新节点坐标
            new_node = (np.array(node_elem.coords) + 
                       self._node_direction(node_elem) * 
                       node_elem.marching_distance)
            
            # 搜索是否有已存在的合适点
            search_radius = self.al * node_elem.marching_distance
            candidates = get_candidate_elements_id(front, space_index, search_radius)
            
            for f_id in candidates:
                candidate = front_dict.get(f_id)
                for node_tmp in candidate.node_elems:
                    if points_equal(node_tmp.coords, new_node.tolist(), 1e-6):
                        new_node_elem = node_tmp
                        existed = True
                        break
            
            if not existed:
                # 创建新节点
                new_node_elem = NodeElementALM(coords=new_node.tolist(), ...)
                new_node_elem.strandline_start_node = node_elem.strandline_start_node
            
            new_cell_nodes.append(new_node_elem)
        else:
            # 使用已推进的对应点
            new_cell_nodes.append(node_elem.corresponding_node)
    
    # 创建新阵面
    alm_front = Front(new_cell_nodes[2], new_cell_nodes[3], -1, "prism-cap", ...)
    new_front1 = Front(new_cell_nodes[0], new_cell_nodes[2], -1, "interior", ...)
    new_front2 = Front(new_cell_nodes[3], new_cell_nodes[1], -1, "interior", ...)
    
    # 创建四边形单元（0-1-3-2 顺序）
    new_cell = Quadrilateral(
        new_cell_nodes[0], new_cell_nodes[1],
        new_cell_nodes[3], new_cell_nodes[2],
        part_name="interior-blayers", idx=self.num_cells
    )
    new_cell.layer = self.ilayer + 1
    
    return new_cell, alm_front, new_front1, new_front2, new_node_generated, new_cell_nodes
```

#### 3.4.3 阵面更新策略

**全局更新**（update_front_list_globally）：
- 在所有部件的 `front_list` 中查找重复阵面
- 若已存在，则从原阵面列表中删除
- 使用 R-tree 空间索引加速查找

**局部更新**（update_front_list_locally）：
- 仅在当前部件的 `front_list` 中查找
- 适用于单部件推进场景

### 3.5 早停条件检查

#### 3.5.1 几何检查（geometric_checker）

```python
def geometric_checker(self, front, new_cell):
    """检查新单元质量，判断是否需要早停"""
    # 1. 单元质量检查
    quality = new_cell.get_skewness()
    if quality < self.quality_threshold:  # 默认 0.005
        return True  # 质量太差，早停
    
    # 2. 层数条件
    full_layer_condition = self.ilayer >= self.full_layers
    
    # 3. 单元尺寸检查
    cell_size = new_cell.get_element_size()
    isotropic_size = self.sizing_system.spacing_at(front.center)
    size_factor = 1.3
    size_condition = cell_size > size_factor * isotropic_size
    
    if size_condition and full_layer_condition:
        return True
    
    # 4. 长宽比检查
    cell_aspect_ratio = new_cell.get_aspect_ratio2()
    aspect_ratio_condition = cell_aspect_ratio < 1.1
    
    if aspect_ratio_condition and full_layer_condition:
        return True
    
    return False
```

**早停条件**：
1. **质量过低**：偏斜度 < `quality_threshold`
2. **尺寸过大**：单元尺寸 > 1.3 × 当地各向同性尺寸，且层数 > `full_layers`
3. **长宽比过小**：长宽比 < 1.1，且层数 > `full_layers`

#### 3.5.2 邻近检查（proximity_checker）

```python
def proximity_checker(self, front, check_fronts):
    """检查新阵面附近是否有其他阵面，避免碰撞"""
    # 安全距离：0.5 倍推进步长
    safe_distance = 0.5 * min(n.marching_distance for n in front.node_elems)
    safe_distance_sq = safe_distance * safe_distance
    
    for new_front in check_fronts:
        # R-tree 搜索候选阵面
        search_radius = self.al * 2.0 * safe_distance
        candidates = get_candidate_elements_id(new_front, space_index, search_radius)
        
        for f_id in candidates:
            candidate = front_dict.get(f_id)
            
            # 跳过共点、共坐标、反方向的阵面
            if candidate 与 new_front 共点：continue
            if candidate 与 new_front 共坐标：continue
            if candidate 在 front 推进方向反方向：continue
            
            # 距离检查
            if fast_distance_check(p0, p1, q0, q1, safe_distance_sq):
                return True  # 距离过近，早停
    
    return False
```

#### 3.5.3 层数差检查（_check_neighbor_layer_difference）

**目的**：避免相邻区域推进层数差异过大导致网格质量下降

**算法**：BFS 搜索通过 interior 阵面相连的所有 prism-cap 阵面

```python
def _check_neighbor_layer_difference(self, front):
    """检查相邻 prism-cap 阵面的层数差"""
    max_layer_diff = 2  # 最大允许层数差
    
    # BFS 搜索相邻 prism-cap
    queue = deque(front.node_elems)
    visited_nodes = set()
    neighbor_layers = set()
    
    while queue:
        current_node = queue.popleft()
        
        for neighbor_front in current_node.node2front:
            if neighbor_front.bc_type != "interior":
                continue
            
            # 找到 interior 阵面的另一个节点
            other_node = ...
            
            # 检查另一个节点所属的 prism-cap
            for other_front in other_node.node2front:
                if other_front.bc_type == "prism-cap":
                    neighbor_layers.add(other_front.layer_count)
            
            # 继续 BFS
            if other_node.hash not in visited_nodes:
                visited_nodes.add(other_node.hash)
                queue.append(other_node)
    
    # 检查层数差
    for neighbor_layer in neighbor_layers:
        if abs(front.layer_count - neighbor_layer) > max_layer_diff:
            return True  # 层数差超限
    
    return False
```

## 4. 多方向推进算法

### 4.1 算法原理

#### 4.1.1 问题背景

在凸角处，传统单方向推进会导致：
1. **网格挤压**：两侧推进方向汇聚，单元严重变形
2. **过早碰撞**：推进距离短，无法生成足够层数
3. **质量下降**：偏斜度大，影响 CFD 计算精度

**解决方案**：在凸角处分裂多个推进方向，每个方向独立推进。

#### 4.1.2 核心思想

1. **虚拟点分裂**：在凸角处创建多个虚拟点，每个虚拟点对应一个推进方向
2. **虚拟阵面连接**：用虚拟阵面连接真实凸点和虚拟点，以及虚拟点之间
3. **独立推进**：每个虚拟点沿各自方向独立推进
4. **串线追踪**：记录从同一凸点出发的所有推进点
5. **后处理消除**：推进完成后，消除虚拟点，恢复为真实点

### 4.2 多方向初始化

#### 4.2.1 初始化流程（initialize_multi_direction）

```python
def initialize_multi_direction(self, front_node_list, ilayer):
    """在第一层为每个凸点创建多个推进方向和虚拟点"""
    if ilayer != 0 or not self.adlayers.multi_direction:
        return

    self.convex_points = []
    for node in front_node_list:

            self.convex_points.append(node)
    
    # 为每个凸点创建虚拟点
    for convex_node in self.convex_points:
        self._create_virtual_points_for_convex_node(convex_node)
```

#### 4.2.2 虚拟点创建（_create_virtual_points_for_convex_node）

**几何示意**：
```
真实凸点 P0，凸角角度θ，分裂为 3 个方向：
  
  方向 2 (虚拟点 V2)
       *
      / 
     /   方向 1 (虚拟点 V1)
    *----*----> 方向 0 (真实点 P0)
   P0    V1
    \
     \
      *
       V2

虚拟阵面连接：
P0 -> V1 (虚拟阵面 1)
V1 -> V2 (虚拟阵面 2)
```

**核心代码**：
```python
def _create_virtual_points_for_convex_node(self, convex_node):
    """为凸点创建虚拟点和虚拟阵面"""
    num_directions = convex_node.num_multi_direction
    multi_directions = convex_node.marching_direction
    
    # 记录真实点
    real_idx = convex_node.idx
    self.node_pair[real_idx] = [convex_node]
    
    # 设置真实点的推进方向为第一个方向
    convex_node.marching_direction = multi_directions[0]
    convex_node.direction_idx = 0
    
    # 创建虚拟点
    base_coords = np.array(convex_node.coords)
    for i in range(1, num_directions):
        virtual_node = self._create_virtual_point(
            coords=base_coords.copy(),
            direction=multi_directions[i],
            real_point_idx=real_idx,
            local_step_factor=convex_node.local_step_factor,
        )
        
        # 添加到推进列表
        self.adlayers.front_node_list.append(virtual_node)
        
        # 记录虚拟点
        convex_node.virtual_points.append(virtual_node)
        self.virtual_points.append(virtual_node)
        self.node_pair[real_idx].append(virtual_node)
    
    # 创建虚拟阵面
    if len(convex_node.virtual_points) > 0:
        # 真实凸点 -> 第一个虚拟点
        virtual_front1 = self._create_virtual_front(
            convex_node, convex_node.virtual_points[0]
        )
        
        # 虚拟点之间连接
        for j in range(len(convex_node.virtual_points) - 1):
            virtual_front = self._create_virtual_front(
                convex_node.virtual_points[j],
                convex_node.virtual_points[j+1]
            )
    
    # 修改右侧阵面：起点更新为最后一个虚拟点
    if right_front is not None:
        last_virtual = convex_node.virtual_points[-1]
        right_front.node_elems[0] = last_virtual
        right_front.node_ids[0] = last_virtual.idx
```

### 4.3 推进方向光滑

#### 4.3.1 方向光滑流程（smooth_advancing_direction）

```python
def smooth_advancing_direction(self, front_node_list, smooth_iterations=3, relax_factor=0.5):
    """光滑推进方向"""
    # 阶段 1：固定特殊节点方向
    fixed_directions = {}
    for node in front_node_list:
        # prism-cap/interior 节点方向固定
        fixed_dir = self._get_prism_cap_interior_direction(node)
        if fixed_dir is not None:
            fixed_directions[node.idx] = fixed_dir
    
    # 阶段 2：迭代光滑
    for iteration in range(smooth_iterations):
        updated_directions = {}
        
        for node in front_node_list:
            # 凹点不参与光滑
            if node.concav_flag:
                continue
            
            # 固定方向的节点不参与光滑
            if node.idx in fixed_directions:
                continue
            
            # 获取邻居节点
            neighbors = node.node2node
            if len(neighbors) < 2:
                continue
            
            # 计算邻居平均方向
            neighbor_dirs = []
            for neighbor in neighbors:
                if neighbor.concav_flag:
                    continue
                neighbor_dir = self._extract_direction(neighbor.marching_direction)
                if neighbor_dir is not None:
                    neighbor_dirs.append(neighbor_dir)
            
            if not neighbor_dirs:
                continue
            
            avg_neighbor_dir = np.mean(neighbor_dirs, axis=0)
            avg_neighbor_dir = self._normalize(avg_neighbor_dir)
            
            # 加权平均
            current_dir = self._extract_direction(node.marching_direction)
            new_dir = (1 - relax_factor) * current_dir + relax_factor * avg_neighbor_dir
            new_dir = self._normalize(new_dir)
            
            updated_directions[node.idx] = new_dir
        
        # 统一更新
        for node in front_node_list:
            if node.idx in updated_directions:
                new_dir = updated_directions[node.idx]
                if isinstance(node.marching_direction, list):
                    node.marching_direction[0] = tuple(new_dir)
                else:
                    node.marching_direction = tuple(new_dir)
```

**关键点**：
1. **凹点排除**：凹点不参与光滑，保持原始方向
2. **固定方向**：prism-cap/interior 节点方向固定
3. **松弛因子**：`relax_factor=0.5`，平衡光滑与保持原方向

### 4.4 步长缩放系数

#### 4.4.1 计算流程（calculate_step_scale_coeff）

```python
def calculate_step_scale_coeff(self, front_node_list, ilayer):
    """计算步长缩放系数"""
    # 清空缓存
    self.step_factor_cache_by_real_node = {}
    
    # 阶段 1：计算真实点步长因子
    for node in front_node_list:
        if node.is_virtual_point:
            continue
        
        # 首层后跳过真实凸点
        if ilayer > 0 and node.convex_flag and not node.is_virtual_point:
            continue
        
        # 获取相邻 wall 阵面
        picked = self._pick_wall_fronts(node)
        if picked is None:
            continue
        
        front1, front2 = picked
        np_dir = self._extract_direction(node.marching_direction)
        
        # 计算投影系数
        coeff1 = np.dot(np.array(front1.normal), np_dir)
        coeff2 = np.dot(np.array(front2.normal), np_dir)
        coeff = (abs(coeff1) + abs(coeff2)) / 2.0
        
        # 调整局部步长因子
        if coeff > 1e-10:
            node.local_step_factor = node.local_step_factor / coeff
        
        # 缓存真实凸点系数
        if node.convex_flag:
            self._cache_step_factor_for_real_node(node.idx, node.local_step_factor)
    
    # 阶段 2：虚拟点步长因子从真实点复制
    if ilayer == 0:
        for node in front_node_list:
            if not node.is_virtual_point:
                continue
            
            real_node = self.get_real_point(node)
            if real_node is not None:
                node.local_step_factor = real_node.local_step_factor
    
    # 阶段 3：串线上的点继承真实点步长
    for node in front_node_list:
        if not node.strandline_start_node:
            continue
        
        start_node = self.get_strandline_start_node(node)
        real_node = self.get_real_point(start_node)
        if real_node:
            node.local_step_factor = self._get_cached_step_factor_for_real_node(real_node.idx)
```

**物理意义**：
- 推进方向与阵面法向夹角越大，投影系数越小
- 步长因子 = 1/coeff，补偿投影损失
- 确保实际推进距离在法向的投影等于理论步长

### 4.6 后处理：虚拟点消除

#### 4.6.1 消除流程（post_process）

```python
def post_process(self):
    """后处理：消除虚拟点"""
    fronts_to_remove = []
    
    for part in self.adlayers.part_params:
        for front in part.front_list:
            node1, node2 = front.node_elems
            
            is_virt1 = getattr(node1, 'is_virtual_point', False)
            is_virt2 = getattr(node2, 'is_virtual_point', False)
            
            if is_virt1 and is_virt2:
                # 两个点都是虚拟点，标记删除
                fronts_to_remove.append(front)
            elif is_virt1:
                # 第一个点是虚拟点，替换为真实点
                real_node = self.get_real_point(node1)
                if real_node:
                    front.node_elems[0] = real_node
                    front.node_ids[0] = real_node.idx
            elif is_virt2:
                # 第二个点是虚拟点，替换为真实点
                real_node = self.get_real_point(node2)
                if real_node:
                    front.node_elems[1] = real_node
                    front.node_ids[1] = real_node.idx
            
            # 检查两点是否重合
            if node1.idx == node2.idx:
                fronts_to_remove.append(front)
    
    # 删除虚拟阵面
    for part in self.adlayers.part_params:
        part.front_list = [f for f in part.front_list if f not in fronts_to_remove]
    
    # 压缩节点（移除未引用的虚拟点）
    self._compact_adlayers_nodes()
```

#### 4.6.2 节点压缩（_compact_adlayers_nodes）

**目的**：移除后处理后不再被引用的虚拟点，重新编号节点

```python
def _compact_adlayers_nodes(self):
    """压缩节点坐标与编号"""
    # 收集被引用的节点编号
    referenced_ids = set()
    
    for part in self.adlayers.part_params:
        for front in part.front_list:
            for node in front.node_elems:
                referenced_ids.add(node.idx)
    
    for cell in self.adlayers.cell_container:
        if hasattr(cell, "node_ids"):
            referenced_ids.update(cell.node_ids)
    
    # 构建新旧编号映射
    old_to_new = {old_id: new_id for new_id, old_id in enumerate(sorted(referenced_ids))}
    
    # 更新单元节点编号
    for cell in self.adlayers.cell_container:
        if hasattr(cell, "node_ids"):
            cell.node_ids = [old_to_new[node_id] for node_id in cell.node_ids]
    
    # 更新节点编号
    for node in all_nodes:
        if node.idx in old_to_new:
            node.idx = old_to_new[node.idx]
    
    # 更新阵面节点编号
    for front in all_fronts:
        front.node_ids = [node.idx for node in front.node_elems]
    
    # 压缩坐标数组
    kept_old_ids = sorted(old_to_new.keys())
    self.adlayers.node_coords = [self.adlayers.node_coords[node_id] for node_id in kept_old_ids]
    self.adlayers.num_nodes = len(old_to_new)
```

### 4.7 终止条件

#### 4.7.1 凸点终止（check_termination）

```python
def check_termination(self, node, ilayer, aspect_ratio, skewness):
    """检查是否满足终止条件"""
    full_layers = self.adlayers.full_layers
    max_layers = self.adlayers.max_layers
    
    # 完整层数内不终止
    if ilayer < full_layers:
        return False
    
    # 超过最大层数
    if ilayer >= max_layers:
        return True
    
    # 偏斜度条件
    if skewness >= 1.0:
        return True
    
    # 凸点终止条件
    if node.convex_flag:
        real_idx = node.idx
        
        # 多方向串线上的点
        if node.strandline_start_node:
            if ilayer >= self.stop_layer_convex[real_idx]:
                return True
        else:
            # 非多方向点，检查质量
            if aspect_ratio < 1.2 or skewness >= 0.75:
                return True
    
    # 凹点终止条件
    if node.concav_flag:
        if ilayer >= self.stop_layer_concav[node.idx]:
            return True
    
    return False
```

### 6.1 单元质量度量

#### 6.1.1 偏斜度（Skewness）

```python
def get_skewness(self):
    """计算四边形偏斜度"""
    # 基于内角计算
    angles = self.get_interior_angles()
    
    # 理想四边形内角为 90°
    ideal_angle = 90.0
    max_deviation = max(abs(angle - ideal_angle) for angle in angles)
    
    # 偏斜度：0-1，0 表示完美，1 表示退化
    skewness = max_deviation / 180.0
    
    return skewness
```

**质量阈值**：
- `skewness < 0.005`：质量极好
- `skewness < 0.5`：质量可接受
- `skewness >= 0.5`：质量较差

```python
def get_aspect_ratio2(self):
```

### 6.2 几何检查函数

#### 6.2.1 点在多边形内（point_in_polygon）

```python
def point_in_polygon(point, polygon):
    """射线法判断点是否在多边形内"""
    x, y = point
    inside = False
    
    n = len(polygon)
    p1x, p1y = polygon[0]
    for i in range(n + 1):
        p2x, p2y = polygon[i % n]
        if y > min(p1y, p2y):
            if y <= max(p1y, p2y):
                if x <= max(p1x, p2x):
                    if p1y != p2y:
                        xinters = (y - p1y) * (p2x - p1x) / (p2y - p1y) + p1x
                    if p1x == p2x or x <= xinters:
                        inside = not inside
        p1x, p1y = p2x, p2y
    
    return inside
```

#### 6.2.2 线段相交（segments_intersect）

```python
def segments_intersect(p1, p2, p3, p4):
    """判断线段 p1-p2 与 p3-p4 是否相交"""
    def ccw(A, B, C):
        return (C[1] - A[1]) * (B[0] - A[0]) > (B[1] - A[1]) * (C[0] - A[0])
    
    return ccw(p1, p3, p4) != ccw(p2, p3, p4) and ccw(p1, p2, p3) != ccw(p1, p2, p4)
```

#### 6.2.3 快速距离检查（fast_distance_check）

```python
def fast_distance_check(p0, p1, q0, q1, safe_distance_sq):
    """快速检查两线段距离是否小于安全距离"""
    # 使用平方距离避免开方
    # 返回 True 表示距离过近
    
    # 检查端点距离
    for p in [p0, p1]:
        for q in [q0, q1]:
            dist_sq = (p[0]-q[0])**2 + (p[1]-q[1])**2
            if dist_sq < safe_distance_sq:
                return True
    
    # TODO: 进一步检查线段间最短距离
    return False
```

### 6.3 空间索引

#### 6.3.1 R-tree 索引

```python
def build_space_index_with_RTree(front_list, front_dict):
    """构建 R-tree 空间索引"""
    space_index = Rtree()
    
    for front in front_list:
        # 计算阵面包络盒
        min_x = min(front.node_elems[0].coords[0], 
                   front.node_elems[1].coords[0])
        max_x = max(front.node_elems[0].coords[0], 
                   front.node_elems[1].coords[0])
        min_y = min(front.node_elems[0].coords[1], 
                   front.node_elems[1].coords[1])
        max_y = max(front.node_elems[0].coords[1], 
                   front.node_elems[1].coords[1])
        
        # 插入 R-tree
        space_index.insert(front.idx, (min_x, min_y, max_x, max_y))
        front_dict[front.idx] = front
    
    return space_index, front_dict
```

#### 6.3.2 候选阵面搜索

```python
def get_candidate_elements_id(front, space_index, search_radius):
    """搜索给定范围内的候选阵面"""
    # 计算查询包络盒
    min_x = min(front.node_elems[0].coords[0], 
               front.node_elems[1].coords[0]) - search_radius
    max_x = max(front.node_elems[0].coords[0], 
               front.node_elems[1].coords[0]) + search_radius
    min_y = min(front.node_elems[0].coords[1], 
               front.node_elems[1].coords[1]) - search_radius
    max_y = max(front.node_elems[0].coords[1], 
               front.node_elems[1].coords[1]) + search_radius
    
    # R-tree 查询
    candidates = list(space_index.intersection((min_x, min_y, max_x, max_y)))
    
    return candidates
```

## 7. 鲁棒性措施

### 7.1 早停机制

**多层级早停判断**：
1. **几何检查失败**：质量、尺寸、长宽比不满足要求
2. **邻近检查失败**：与附近阵面距离过近
3. **层数差超限**：相邻区域层数差 > 2
4. **零长度阵面**：阵面长度 < 1e-12

**早停处理**：
- 设置 `early_stop_flag = True`
- 阵面保留到下一层（prism-cap 类型）
- 不再生成新单元

### 7.2 虚拟阵面处理

**零长度虚拟阵面推进**（_advance_virtual_front）：

```python
def _advance_virtual_front(self, front, new_interior_list, new_prism_cap_list):
    """处理零长度虚拟阵面推进"""
    if front.length >= 1e-12:
        return False
    
    node1, node2 = front.node_elems
    is_virt1 = getattr(node1, "is_virtual_point", False)
    is_virt2 = getattr(node2, "is_virtual_point", False)
    
    if not (is_virt1 or is_virt2):
        debug(f"阵面{front.node_ids}长度为 0 且非虚拟阵面，跳过推进")
        return True
    
    # 推进生成新点
    node1_new = self._get_or_create_corresponding_node(node1)
    node2_new = self._get_or_create_corresponding_node(node2)
    
    # 检查新点是否重合
    if points_equal(node1_new.coords, node2_new.coords, 1e-12):
        debug(f"阵面{front.node_ids}推进后新点重合，跳过")
        return True
    
    # 获取真实点
    node1_real = self.multi_direction_manager.get_real_point(node1)
    node2_real = self.multi_direction_manager.get_real_point(node2)
    
    # 构建有效点序列（去重）
    ordered_nodes = [node1_real, node2_real, node2_new, node1_new]
    cell_nodes = []
    seen = set()
    for node in ordered_nodes:
        if node.idx not in seen:
            seen.add(node.idx)
            cell_nodes.append(node)
    
    # 创建单元（四边形或三角形）
    if len(cell_nodes) == 4:
        new_cell = Quadrilateral(...)
    elif len(cell_nodes) == 3:
        new_cell = Triangle(...)
    else:
        warning(f"有效点不足 3 个，无法构成单元")
        return True
    
    # 更新阵面
    virtual_alm_front = Front(node1_new, node2_new, -1, "prism-cap", ...)
    virtual_new_front1 = Front(node1, node1_new, -1, "interior", ...)
    virtual_new_front2 = Front(node2_new, node2, -1, "interior", ...)
    
    self._finalize_new_fronts(front, [virtual_alm_front, virtual_new_front1, virtual_new_front2], ...)
    
    return True
```

### 7.3 节点重编号与压缩

**问题**：后处理消除虚拟点后，节点编号不连续，存在大量空洞

**解决方案**：
1. 收集所有被引用的节点编号
2. 构建新旧编号映射
3. 更新所有拓扑引用
4. 压缩坐标数组

**优势**：
- 减少内存占用
- 提高数据局部性
- 便于后续处理

### 7.4 Match 边界处理

**Match 边界**：两个部件的边界需要匹配连接

**处理流程**：
1. **识别 match 边界**：`is_match = True` 的 connector
2. **重新离散化**：`rediscretize_conn_to_match_wall()`
3. **阵面同步**：确保两侧节点一一对应
4. **推进方向指定**：使用 `marching_vector`

## 8. 算法流程总结

### 8.1 完整算法流程

```
开始
  ↓
[预处理]
├─ 读取部件参数
├─ 初始化阵面列表
├─ Match 边界处理
│   └─ 重新离散化 match connector
└─ 节点重编号
  ↓
[逐部件推进]
  ↓
设置当前部件参数
  ↓
[单层推进循环] (ilayer = 0 to max_layers-1)
├─ 准备几何信息
│   ├─ compute_front_geometry()
│   ├─ compute_point_normals()
│   └─ laplacian_smooth_normals()
├─ 多方向流程（若开启）
│   ├─ initialize_multi_direction() [仅首层]
│   ├─ smooth_advancing_direction()
│   └─ calculate_step_scale_coeff()
├─ 计算推进距离
│   └─ calculate_marching_distance()
├─ 阵面推进
│   ├─ 遍历 prism-cap 阵面
│   ├─ 创建新单元和阵面
│   ├─ 几何检查
│   ├─ 邻近检查
│   ├─ 层数差检查
│   └─ 更新数据结构
└─ 更新阵面列表
  ↓
[后处理]
├─ 消除虚拟点
│   ├─ 虚拟阵面转真实阵面
│   └─ 删除无效虚拟阵面
├─ 节点压缩
│   ├─ 收集引用节点
│   ├─ 重新编号
│   └─ 压缩坐标数组
└─ 构建非结构网格
  ↓
输出网格和边界阵面
  ↓
结束
```

### 8.2 关键参数推荐值

| 参数名 | 推荐值 | 说明 |
|-------|-------|------|
| `first_height` | 0.001-0.01 | 第一层高度，根据 y+ 要求确定 |
| `growth_rate` | 1.1-1.3 | 增长率，推荐 1.2 |
| `max_layers` | 15-30 | 最大层数 |
| `full_layers` | 8-15 | 完整层数（不进行早停判断） |
| `multi_direction` | True | 凸角处开启多方向 |
| `relax_factor` | 0.2-0.5 | 方向光滑松弛因子 |
| `smooth_iterions` | 3 | Laplacian 光滑迭代次数 |
| `quality_threshold` | 0.005 | 单元偏斜度阈值 |
| `al` | 3.0 | 邻近检查搜索范围系数 |

### 8.3 性能优化建议

1. **空间索引**：使用 R-tree 加速邻近搜索
2. **早停判断**：尽早排除不合格阵面，减少无效计算
3. **节点压缩**：后处理阶段压缩节点，减少内存
4. **批量更新**：阵面更新采用批量方式，减少重复查找
5. **调试级别**：`debug_level=0` 关闭调试输出，提高运行速度
